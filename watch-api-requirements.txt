CHAYNOVA Watch API routes required by the two HTML pages

GET    /watch
Returns a JSON array:
[
  {
    "id": "1",
    "emoji": "🎬",
    "title": "Coming Soon",
    "description": "New CHAYNOVA videos are coming soon...",
    "button_text": "Watch Soon",
    "button_url": "https://example.com/video",
    "position": 1,
    "created_at": "2026-07-10T14:00:00.000Z",
    "updated_at": null
  }
]

POST   /watch
Authorization: Bearer ADMIN_TOKEN
Content-Type: application/json

PUT    /watch/:id
Authorization: Bearer ADMIN_TOKEN
Content-Type: application/json

DELETE /watch/:id
Authorization: Bearer ADMIN_TOKEN

Suggested D1 table:

CREATE TABLE IF NOT EXISTS watch_cards (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    emoji TEXT NOT NULL DEFAULT '🎬',
    title TEXT NOT NULL,
    description TEXT NOT NULL,
    button_text TEXT NOT NULL DEFAULT 'Watch',
    button_url TEXT NOT NULL,
    position INTEGER NOT NULL DEFAULT 0,
    created_at TEXT NOT NULL,
    updated_at TEXT
);
